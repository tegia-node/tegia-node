# Tasks: модель статусов

Эта модель задает единые статусы для задач в `docs/tasks`.

## Статусы

1. **Запланировано**
   - Работа описана, выполнение не начато.

2. **В процессе**
   - Работа начата, но не завершена.

3. **Выполнено**
   - Работа завершена и результат внедрен.

4. **Отложено**
   - Выполнение приостановлено или перенесено на неопределенный срок.

5. **Отменено**
   - Работа признана неактуальной и закрыта без реализации.

## Примечания

- Статус указывается строкой `- Статус: ...` в начале файла задачи.
- При необходимости можно уточнять контекст в тексте задачи (например, `Phase 1 выполнен`).
