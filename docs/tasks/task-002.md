# Task-002: Реализация внешних WS маршрутов

- Статус: Выполнено
- Дата: 2026-03-26
- Основание: `docs/adr/ADR-0002-ws-route-registration-model.md`
- Цель: реализовать поддержку внешних WS маршрутов и затем зафиксировать это в документации.

## 1. Объем работ

Входит:
1. Подтвердить, что для внешнего `route.actor` достаточно существующего `ADD_WS_ROUTE` (без `add_action`) и зафиксировать это как основной путь.
2. Обновить SDK-документацию о WS маршрутах после подтверждения.
3. Добавить пример маршрута на внешний actor (`actor = "crm/client"`, `action = "/sync"`).

Не входит:
1. Миграция существующих конфигураций.

## 2. Точки изменения по файлам

### 2.1 `include/tegia/actors/type.h`

Подтвердить, что:
1. `ADD_WS_ROUTE` допускает `actor` как имя внешнего актора.
2. Валидация не требует локального `add_action` для внешних маршрутов.
3. `route.action` должен начинаться с `/` (как и сейчас).

### 2.2 `include/tegia/docs/ws-routing-contract.md`

Место вставки: раздел **1. Как объявлять route**, сразу после базового примера `ADD_WS_ACTION_ROUTE`.

Добавить нормы (после подтверждения):
1. `route.actor` может быть `{/this}` или именем любого актора, включая удаленный.
2. Регистрация `action` для внешнего актора локально не требуется и не должна выполняться.
3. `ADD_WS_ACTION_ROUTE` нельзя использовать для внешнего актора, так как он регистрирует локальный action.

Добавить пример:
1. Маршрут `POST` с `actor` равным имени другого актора.
2. Явно показать, что `action` принадлежит другому actor.
3. Использовать `ADD_WS_ROUTE` с `actor = "crm/client"` и `action = "/sync"`.

### 2.3 `include/tegia/docs/quickstart-new-actor.md`

Место вставки: раздел **2. Зарегистрируй type в `_init_type`**, после строки про `ADD_WS_ACTION_ROUTE`.

Добавить краткое примечание (после подтверждения):
1. Можно указывать `actor` не равный `{/this}`.
2. Это допустимо для межнодовых вызовов.
3. Для внешнего актора использовать `ADD_WS_ROUTE`, а не `ADD_WS_ACTION_ROUTE`.

## 3. Критерии приемки

1. Подтверждено, что для внешнего маршрута достаточно `ADD_WS_ROUTE` без `add_action`.
2. Валидация `route.actor` допускает внешние имена (строка).
3. В `include/tegia/docs/ws-routing-contract.md` есть текстовое правило про внешний actor.
4. В `include/tegia/docs/ws-routing-contract.md` есть пример маршрута на внешний actor.
5. В `include/tegia/docs/quickstart-new-actor.md` есть краткое примечание о `actor != {/this}`.
