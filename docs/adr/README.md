# ADR: модель статусов

Эта модель задает единые статусы для всех ADR в `docs/adr`.

## Обязательные статусы

1. **Черновик**
   - Идея сформулирована, решение не утверждено.

2. **Утвержден**
   - Решение принято, можно планировать и начинать реализацию.

3. **В процессе реализации**
   - Реализация начата, но не завершена.

4. **Реализовано**
   - Решение полностью внедрено и работает в runtime.

## Дополнительные статусы (по необходимости)

1. **Отклонен**
   - Решение рассмотрено и не принято.

2. **Отложен**
   - Решение принято/обсуждено, но реализация отложена.

3. **Заменен**
   - ADR больше не актуален, есть новый ADR‑замена (с ссылкой).

4. **Устарел**
   - Решение работает, но больше не рекомендуется и будет выведено.

## Примечания

- Статус указывается в заголовке ADR строкой `- Статус: ...`.
- Для поэтапных внедрений допускается добавлять пояснение в тексте ADR (например, `Phase 1 выполнен`).
